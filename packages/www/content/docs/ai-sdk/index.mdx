`@castaisdk/ai-sdk` provides Vercel AI SDK tools and React testing components for castAI payment flows.

Use it when an agent needs to request protected resources that require:

- x402 payment over HTTP.
- MPP payment over HTTP.
- Casper CSPR settlement.

## Install

<PackageInstall packages="@castaisdk/ai-sdk ai @castaisdk/x402 @castaisdk/mpp casper-js-sdk" />

## Exports

```ts
import {
  createCastaiAgent,
  createCastaiAgentTools,
  createCasperMppFetch,
  createCasperX402Fetch,
  generateCastaiText,
  CastaiResourceError,
  llm,
} from "@castaisdk/ai-sdk";
```

React components are exported separately:

```tsx
import { PaymentTester } from "@castaisdk/ai-sdk/react";
```

## What the tools do

| Tool | Purpose |
| --- | --- |
| `payX402Resource` | Fetch an x402-protected resource using a configured paid x402 fetcher |
| `payMppResource` | Fetch an MPP-protected resource using a configured MPP fetcher |

The tools do not simulate payment. If a payment fetcher is missing, the tool throws a setup error.

## Higher level helpers

| Helper | Purpose |
| --- | --- |
| `createCastaiAgent` | Builds x402/MPP fetchers, AI SDK tools, and a default system instruction from one config |
| `generateCastaiText` | Calls Vercel AI SDK `generateText` with castAI tools and multi-step tool calling |
| `llm.text` | Formats paid resource responses as plain text for model context |

## Resource request controls

`fetchResource` validates resource URLs and methods before calling the configured paid fetcher. It also accepts timeout and error-handling options:

```ts
import { CastaiResourceError, fetchResource } from "@castaisdk/ai-sdk";

try {
  const response = await fetchResource(
    x402Fetch,
    {
      method: "GET",
      url: "https://api.example.com/weather",
    },
    {
      timeoutMs: 15_000,
      throwOnError: true,
    }
  );
} catch (error) {
  if (error instanceof CastaiResourceError) {
    console.error(error.response.status, error.response.body);
  }
}
```

Responses with `application/json` and `application/*+json` content types are parsed as JSON. Other response bodies are returned as text.
